# Why MetricShell

---
[Prev: Existing solutions](existing-solutions.md)

---

MetricShell is not intended to replace Prometheus, Pushgateway, OpenTelemetry, sidecar exporters or textfile collectors.

The project explores an alternative deployment model for workloads where existing approaches introduce unnecessary
application changes, additional infrastructure or environment-specific deployment patterns.

The objective is to evaluate whether a reusable runtime layer can provide a simpler operational model while remaining
fully compatible with the Prometheus pull ecosystem.

The detailed comparison with existing approaches is documented in [Existing solutions](existing-solutions.md).
